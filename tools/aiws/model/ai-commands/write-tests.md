# write-tests

## Description
Escrever testes antes ou durante a implementacao.

## Steps
1. Ler o Plan da task para entender o que sera testado
2. Consultar AGENTS.md secao Testing Rules para seguir as convencoes
3. Identificar tipo de teste necessario (unit, integration, e2e)
4. Escrever testes seguindo naming convention: should ... when ...
5. Verificar que testes estao co-locados quando pratico
6. Executar testes para confirmar que falham antes da implementacao (red phase)

## Output
Arquivos de teste criados e executados.
